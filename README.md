# Atividade 1 de API e Web Services - Sistema Bancário 

Desenvolva um sistema modular para gerenciamento de uma instituição bancária. O sistema deve ser capaz de gerenciar dados de clientes, contas-
correntes, cartões de crédito e empréstimos. O projeto deve ser implementado por meio de quatro módulos distintos e cada um contendo Persistência, Regras e Domínio.

As funcionalidades dos módulos devem ser incorporadas a um projeto distinto e gerar um único programa executável. Cada módulo deve definir as interfaces de
suas respectivas camadas. A implementação das interfaces não precisa ter efeito, ou seja, devem ser meramente informativas.
Os projetos devem ser gerenciados por sistemas de controle de versão GIT. A evolução do sistema, ou seja, as Requisições de Modificação (RM) dos módulos
(correções, melhorias, novas funcionalidades, etc..) devem ser registradas em arquivo de evolução de sistema, e suas respectivas versões geradas a partir da
estratégia de versionamento semântico. O fechamento de versões deve ser realizado através da criação de uma etiqueta (tag) no repositório Git.

### A evolução do sistema deve seguir a ordem estabelecida a seguir:

* 1. Fechamento da versão 1.0.0 - Todos os módulos.
* 2. RM-1: Correção - módulo clientes.
* 3. RM-2: Correção - módulo cartão de crédito.
* 4. RM-3: Nova Funcionalidade – módulo empréstimo.
* 5. Entrega e implantação das novas versões.
* 6. RM-4: Correção - módulo clientes.
* 7. RM-5: Correção - módulo clientes.
* 1. RM-6: Nova Funcionalidade - módulo clientes.
* 2. Entrega e implantação das novas versões.
* 3. RM-7: Nova Funcionalidade - módulo cartão de crédito.
* 4. RM-8: Correção - módulo cartão de crédito.
* 5. RM-9: Nova Funcionalidade – módulo empréstimo.
* 6. Entrega e implantação das novas versões.
